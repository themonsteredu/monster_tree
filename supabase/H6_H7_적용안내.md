## H-6 — `supabase/migrations/0042_game_center.sql`

- 그동안 코드에 없던 게임센터 테이블(`game_plays`, `game_rankings`)과 오늘 플레이 횟수
  함수(`get_today_play_count`)를 **정식 파일로 복원**했습니다.
- 운영 DB에 이미 있으면 `create ... if not exists` 라 **안전하게 건너뜁니다.**
- 적용: Supabase → SQL Editor 에 `0042_game_center.sql` 붙여넣고 Run.
- 효과: 이제 저장소만으로 DB 구조를 재현할 수 있습니다(새 환경/복구 시 게임 기능 안 깨짐).
- 게임 랭킹은 읽기 공개(리더보드), 원시 플레이 기록(`game_plays`)은 서버 전용으로 잠급니다.

> 참고: 몬스터 진화 임계치/이미지 시드 데이터는 이 파일 범위가 아닙니다(별도 시드).

## H-7 — `supabase/migrations/0043_quiz_antifarm.sql` + 코드

- **문제**: 퀴즈 제출에 하루 제한 검사가 없어, 정답을 아는 채로 반복 제출하면
  "올클" 포인트(+1)를 하루에 여러 번 받을 수 있었습니다.
- **코드 수정**(이미 반영): 제출 시 "오늘 이미 포인트 받았는지" 확인 후 중복 지급 차단 +
  동시 제출(race) 충돌 시 포인트 없이 기록만.
- **SQL**(0043): "학생 × KST 날짜 당 포인트 1건"을 DB가 강제하는 부분 unique 인덱스 추가.
  이게 동시 제출까지 막는 최종 백스톱입니다.
  - 적용 전 과거 중복 포인트 행을 자동 정리(가장 이른 1건만 유지)하므로 인덱스 생성이 실패하지 않습니다.
- 적용: 코드(이 브랜치)를 배포한 뒤, Supabase → SQL Editor 에 `0043_quiz_antifarm.sql` 붙여넣고 Run.
- 적용 후 확인: 학생이 퀴즈 올클 시 하루 1포인트만 적립되는지, 반복 제출해도 추가 적립 안 되는지.

> 정답이 화면에 미리 보이는 즉시피드백 UX는 유지했습니다. 하루 1포인트가 DB로 보장되므로
> 정답을 알든 모르든 farming(중복 적립)은 불가능합니다.
