# 🚨 themonster.kr 긴급대응 매뉴얼 (비개발자용)

> **이 문서는 누구를 위한 건가요?**
> 개발을 모르는 운영자가, 사이트에 문제가 생겼을 때
> **개발자 없이 스스로 1차 대처**를 할 수 있도록 만든 응급처치 설명서입니다.
> 컴퓨터를 잘 몰라도 적힌 순서대로 클릭만 따라 하면 됩니다.

---

## 📚 먼저 알아두면 좋은 기본 용어 (딱 4개만)

우리 사이트는 아래 세 곳이 함께 일해서 돌아갑니다.
이 셋 중 하나라도 문제가 생기면 사이트가 안 보일 수 있어요.

| 이름 | 쉽게 말하면 | 하는 일 |
|------|------------|---------|
| **Vercel (버셀)** | 사이트가 사는 집 | 사람들이 themonster.kr에 들어오면 화면을 보여주는 곳 |
| **Supabase (수파베이스)** | 데이터 창고 | 회원 정보, 글, 사진 같은 **데이터를 보관**하는 곳 |
| **인터넷 / 와이파이** | 집까지 가는 길 | 내 컴퓨터가 인터넷에 연결돼 있어야 사이트가 보임 |

- **배포(Deploy, 디플로이)** = "사이트를 새로 올리는 것". 새 버전으로 갈아끼우는 작업이라고 생각하면 됩니다.
- **로그(Log, 로그)** = "컴퓨터가 남긴 일기장". 무슨 일이 있었는지 기록돼 있어요. 오류 원인을 찾을 때 봅니다.

> 💡 한 줄 요약: **화면이 안 보이면 Vercel, 데이터가 안 보이면 Supabase, 나만 안 되면 내 인터넷** 문제일 확률이 높습니다.

## ⚡ 0단계: 무슨 일이 생기든 가장 먼저 할 것 (30초 점검)

당황하지 말고 아래 4가지를 순서대로 해보세요. 이것만으로 절반은 해결됩니다.

### ✅ 점검 1. 진짜 사이트가 죽은 건지, 나만 그런 건지 확인하기

가장 흔한 착각이 "사이트가 죽었다!" 인데 알고 보면 **내 컴퓨터/브라우저만** 그런 경우가 많아요.

1. **새로고침** 해보기
   - 키보드에서 `F5` 키를 누르거나, 주소창 왼쪽의 **동그란 화살표 모양(↻)** 버튼을 클릭.
2. **강력 새로고침** (저장된 옛날 화면을 지우고 새로 불러오기)
   - Windows: `Ctrl` 키와 `F5` 키를 **동시에** 누르기
   - Mac: `Cmd`(⌘) 키와 `Shift` 키와 `R` 키를 **동시에** 누르기
3. **다른 기기로 접속**해 보기
   - 컴퓨터에서 안 되면 → **휴대폰**으로 themonster.kr 들어가 보기
   - 휴대폰에서도 안 되면 → 진짜 문제일 가능성이 큼 (아래 증상별 대처로)
   - 휴대폰에서는 잘 되면 → **내 컴퓨터/와이파이 문제** (사이트는 멀쩡함!)
4. **다른 사이트(예: naver.com)**가 열리는지 확인
   - 네이버도 안 열리면 → **내 인터넷이 끊긴 것**. 와이파이를 껐다 켜거나 통신사 문제.

### ✅ 점검 2. 외부 도구로 "전 국민이 다 안 되는지" 확인하기

내 기기 문제가 아니라는 게 확실해지면, 아래 사이트에서 객관적으로 확인:

- 👉 **https://downforeveryoneorjustme.com**
  접속 후 칸에 `themonster.kr` 을 입력하고 확인 버튼.
  - "It's just you" = 나만 안 됨 (내 문제)
  - "It's down for everyone" = 다 안 됨 (진짜 사이트 문제)

> 여기까지 했는데도 사이트가 진짜 안 된다면, 아래 **증상별 대처법**으로 갑니다.

## 🩺 증상별 대처법

> 사용법: 내 상황과 가장 비슷한 증상을 찾아 그 부분만 보세요.

---

### 🔴 증상 A. "사이트가 아예 안 열림" (흰 화면 / 빙글빙글 / 연결 안 됨)

**무슨 뜻?** themonster.kr 주소를 쳤는데 아무것도 안 나오거나, 계속 로딩만 되거나, "연결할 수 없음" 같은 메시지가 뜸.

**👉 먼저 의심할 곳: Vercel (사이트가 사는 집)**

**확인 순서:**

1. **0단계 점검**을 먼저 했는지 확인 (나만 그런 건 아닌지)
2. **Vercel 서비스 자체에 문제가 있는지 확인**
   - 👉 **https://www.vercel-status.com** 접속
   - 화면 맨 위에 초록색으로 **"All Systems Operational"** 이라고 적혀 있으면 → Vercel은 정상
   - 빨강/주황색 글씨나 "Outage", "Degraded" 가 보이면 → **Vercel 회사 쪽 장애**. 우리가 할 수 있는 건 없고 복구를 기다려야 함 (보통 몇십 분~몇 시간)
3. **우리 사이트 배포 상태 확인** (Vercel 계정에 로그인할 수 있다면)
   - 👉 **https://vercel.com** 접속 → 우측 상단 **Login(로그인)** 클릭 → 로그인
   - 로그인 후 프로젝트 목록에서 **monster-site** (또는 themonster 관련 이름) 클릭
   - 왼쪽/상단 메뉴에서 **Deployments(디플로이먼츠)** 클릭
   - 맨 위(가장 최근) 항목의 동그란 표시 색깔을 봅니다:
     - 🟢 **초록색 (Ready)** = 정상적으로 올라가 있음
     - 🔴 **빨간색 (Error / Failed)** = 최근에 사이트 올리다가 **실패**함 → 이게 원인일 수 있음
     - 🟡 **노란색 (Building)** = 지금 한창 올리는 중 → 5분쯤 기다렸다 새로고침

**🛠️ 내가 직접 할 수 있는 것:**

- 최근 배포가 🟢 초록인데도 사이트가 안 열린다면 → **잠깐 기다렸다(5분) 다시 시도**. 일시적 현상일 수 있음.
- 가장 최근 배포가 🔴 빨강이라면 → **바로 직전의 🟢 초록색 배포로 되돌리기**(롤백):
  1. Deployments 목록에서 **초록색(Ready)이면서 가장 최근**인 항목을 찾기
  2. 그 항목 오른쪽 끝의 **점 세 개(⋯)** 버튼 클릭
  3. **"Promote to Production"** 또는 **"Redeploy(재배포)"** 클릭
  4. 안내창이 뜨면 확인/Continue 눌러서 진행
  5. 몇 분 뒤 themonster.kr 새로고침해서 살아났는지 확인

**🆘 반드시 도움을 요청해야 하는 것:**

- 위 롤백을 해도 안 살아남
- Deployments 화면 자체가 안 보이거나 로그인이 안 됨
- 빨간색 오류가 났는데 뭘 눌러야 할지 모르겠음
→ 이럴 땐 무리하지 말고 **아래 [클로드에게 도움 요청하기] 방식**으로 캡처해서 전달하세요.

### 🟠 증상 B. "특정 페이지만 오류" (다른 페이지는 멀쩡한데 한 곳만 깨짐)

**무슨 뜻?** 메인은 잘 열리는데, 예를 들어 '강의 목록' 페이지나 '마이페이지'만 오류 화면이 뜸. 화면에 `404`, `500`, `Something went wrong` 같은 글씨가 보일 수 있음.

**👉 먼저 의심할 곳: 그 페이지 자체 / Supabase(데이터)**

**확인 순서:**

1. **그 페이지 주소를 정확히 적어두기** (주소창 전체를 복사). 나중에 설명할 때 꼭 필요해요.
2. 화면에 보이는 **숫자/영어 메시지를 그대로 적거나 캡처**:
   - **404** = "그런 페이지가 없어요" (주소가 틀렸거나, 페이지가 사라짐)
   - **500** = "사이트 안쪽에서 문제가 생겼어요" (데이터나 코드 문제일 가능성)
   - **403** = "권한이 없어요" (로그인 상태나 권한 문제)
3. 데이터가 들어가는 페이지(목록, 게시판 등)라면 → **Supabase**도 의심
   - 👉 **https://status.supabase.com** 접속해서 초록색(정상)인지 확인

**🛠️ 내가 직접 할 수 있는 것:**

- 주소를 잘못 들어간 건 아닌지 확인 (오타, 옛날 링크)
- 메인 화면에서 메뉴를 눌러 정상 경로로 다시 들어가 보기
- 강력 새로고침(`Ctrl`+`F5`)

**🆘 반드시 도움을 요청해야 하는 것:**

- **500 오류**가 뜨는 경우 → 거의 항상 개발/데이터 쪽 문제라 직접 고치기 어려움
- 특정 페이지가 계속 깨지는 경우
→ **페이지 주소 + 오류 화면 캡처**를 챙겨서 도움 요청 (아래 방식 참고)

---

### 🟡 증상 C. "사이트가 너무 느림" (열리긴 하는데 엄청 답답함)

**무슨 뜻?** 페이지가 뜨긴 하는데 한참 걸리거나, 버튼을 눌러도 반응이 느림.

**👉 먼저 의심할 곳: 내 인터넷 → Vercel/Supabase 순서**

**확인 순서:**

1. **내 인터넷 속도부터 확인** (제일 흔한 원인)
   - 다른 사이트(네이버, 유튜브)도 다 같이 느린가요? → **내 인터넷 문제**
   - 다른 곳은 빠른데 우리 사이트만 느린가요? → 사이트 쪽 문제일 수 있음
2. **휴대폰 데이터(와이파이 끄고)**로 접속해서 비교
   - 휴대폰에선 빠르면 → 내 와이파이/공유기 문제
3. Vercel과 Supabase 상태 페이지에 "Degraded Performance(성능 저하)" 표시가 있는지 확인
   - 👉 https://www.vercel-status.com
   - 👉 https://status.supabase.com

**🛠️ 내가 직접 할 수 있는 것:**

- 와이파이 공유기 전원을 **껐다가 30초 후 다시 켜기**
- 브라우저에 켜둔 탭이 너무 많으면 정리하기
- 브라우저 종료 후 다시 켜기

**🆘 반드시 도움을 요청해야 하는 것:**

- 모든 사람이 며칠째 계속 느림 (일시적 현상이 아님)
- 특정 페이지만 유독 매번 느림
→ "언제부터 / 어느 페이지가 / 얼마나 느린지"를 메모해서 전달

### 🟢 증상 D. "로그인이 안 됨"

**무슨 뜻?** 아이디/비밀번호를 넣어도 로그인이 안 되거나, 로그인하면 바로 튕기거나, "인증 오류" 같은 메시지가 뜸.

**👉 먼저 의심할 곳: Supabase (로그인/회원 기능을 담당)**

**확인 순서:**

1. **나만 그런지 확인** — 다른 계정이나 다른 사람도 로그인 안 되는지 물어보기
2. 비밀번호를 잘못 친 건 아닌지 (한/영 전환, `Caps Lock` 켜짐 확인)
3. **Supabase 상태 확인**
   - 👉 **https://status.supabase.com** 접속 → 초록색(Operational)인지 확인
   - 특히 **"Auth(인증)"** 항목이 정상인지 보기. 여기 빨간불이면 로그인 기능 장애
4. 브라우저에 저장된 옛날 로그인 정보가 꼬였을 수 있음 → **시크릿 창**으로 시도
   - Windows: `Ctrl`+`Shift`+`N` (크롬) 으로 시크릿 창 열고 로그인 해보기
   - 시크릿 창에서 되면 → 내 브라우저 캐시 문제 (아래 캐시 지우기 참고)

**🛠️ 내가 직접 할 수 있는 것:**

- 비밀번호 재설정("비밀번호를 잊으셨나요?") 시도
- 시크릿 창 / 다른 브라우저로 시도
- 브라우저 캐시·쿠키 지우기 (맨 아래 [부록] 참고)

**🆘 반드시 도움을 요청해야 하는 것:**

- **모든 사람이** 로그인 안 됨 → 인증 시스템 장애 (직접 못 고침)
- Supabase의 Auth 항목에 빨간불 → 복구 기다리거나 개발자 호출
- "특정 회원만 로그인 안 됨" 이 반복됨

### 🔵 증상 E. "데이터가 안 보임" (목록이 텅 비거나, 글/사진이 사라짐)

**무슨 뜻?** 화면은 멀쩡히 열리는데 안에 있어야 할 내용(강의 목록, 게시글, 회원 정보 등)이 텅 비어 보임.

**👉 먼저 의심할 곳: Supabase (데이터 창고)**

**확인 순서:**

1. **강력 새로고침** (`Ctrl`+`F5`) — 화면만 옛날 걸 보고 있을 수 있음
2. **Supabase 상태 확인**
   - 👉 **https://status.supabase.com** → 특히 **"Database(데이터베이스)"** 항목이 초록색인지 확인
   - 빨간불이면 → 데이터 창고 장애. 복구를 기다려야 함
3. 데이터가 **전부 안 보이는지**, **일부만 안 보이는지** 구분해서 메모
4. (Supabase 계정 접근 가능하면) → 👉 **https://supabase.com** 로그인 후 프로젝트 → 왼쪽 메뉴 **Table Editor(테이블 에디터)** 에서 데이터가 실제로 있는지 눈으로 확인

**🛠️ 내가 직접 할 수 있는 것:**

- 강력 새로고침, 다른 기기로 확인
- Supabase 상태 페이지 확인

**🆘 반드시 도움을 요청해야 하는 것:**

- 데이터가 **실제로 사라진 것 같음** (창고를 봤는데 비어 있음)
- 일부 데이터만 이상하게 안 나옴
→ ⚠️ **데이터 문제는 함부로 건드리면 안 됩니다.** 절대 Supabase에서 무언가를 지우거나 수정하지 말고, 즉시 도움을 요청하세요.

문제가 생겼을 때 "지금 그 회사 자체에 문제가 있는지" 알려주는 공식 페이지입니다.
초록색/정상(Operational)이면 그쪽은 멀쩡한 것, 빨강/주황이면 그쪽 장애입니다.

| 무엇을 확인 | 주소 | 이렇게 읽으세요 |
|------------|------|----------------|
| **Vercel** (사이트 집) | https://www.vercel-status.com | "All Systems Operational" = 정상 |
| **Supabase** (데이터 창고) | https://status.supabase.com | 항목별 초록불 = 정상 |
| **우리 사이트가 진짜 죽었나** | https://downforeveryoneorjustme.com | `themonster.kr` 입력해서 확인 |
| **Vercel 관리 화면** | https://vercel.com | 로그인 후 배포 상태 확인 |
| **Supabase 관리 화면** | https://supabase.com | 로그인 후 데이터 확인 |

> 💡 **이 5개 주소를 브라우저 즐겨찾기(북마크)에 미리 저장해두세요.**
> 즐겨찾기 방법: 그 사이트에 들어간 상태에서 주소창 오른쪽 끝의 **별(☆) 모양**을 클릭.

## 🆘 가장 흔한 오류 5가지 + 클로드에게 설명하는 법

오류가 나면 직접 고치려 하기보다, **정확한 정보를 모아서 클로드(저)에게 전달**하는 게 가장 빠르고 안전합니다.
아래는 자주 보는 오류와, 각각 무슨 뜻인지 + 어떻게 전달하면 되는지입니다.

### 흔한 오류 1. `404 - This page could not be found`
- **뜻:** "그런 페이지가 없어요." 주소가 틀렸거나 페이지가 사라짐.
- **전달법:** "이 주소에서 404가 떠요" 하고 **주소창 전체 + 화면 캡처**를 보내기.

### 흔한 오류 2. `500 - Internal Server Error` (또는 `Application error`)
- **뜻:** "사이트 안쪽에서 문제가 생겼어요." 보통 코드나 데이터 문제. **직접 못 고침.**
- **전달법:** **어느 페이지에서** 떴는지 주소 + 캡처 + "언제부터 그런지"를 보내기.

### 흔한 오류 3. `This Serverless Function has crashed` / `FUNCTION_INVOCATION_FAILED`
- **뜻:** "사이트 뒤에서 돌아가는 작은 프로그램이 멈췄어요." Vercel 화면에서 잘 나옴.
- **전달법:** 화면 캡처 + 그 페이지 주소. (가능하면 Vercel 로그도 — 아래 [로그 복사법] 참고)

### 흔한 오류 4. 로그인/인증 오류 `Invalid login credentials` / `Auth session missing`
- **뜻:** "로그인 정보가 안 맞거나 인증이 풀렸어요." Supabase 인증 관련.
- **전달법:** "나만 그런지/다 그런지" + Supabase status의 Auth 상태 + 캡처를 보내기.

### 흔한 오류 5. 데이터 연결 오류 `Failed to fetch` / `Network Error` / 목록이 텅 빔
- **뜻:** "데이터 창고와 연결이 안 됐어요." Supabase 또는 인터넷 문제.
- **전달법:** Supabase status의 Database 상태 + 어느 화면인지 + 캡처를 보내기.

## 📸 [중요] 오류를 "어디서 복사해서 어떻게 전달"하나요?

오류를 잘 전달할수록 빨리 해결됩니다. 아래 3가지를 챙겨주세요.

### 1) 화면 통째로 캡처하기 (가장 쉬움, 꼭 하기)
- **Windows:** 키보드의 `Windows` 키 + `Shift` + `S` 를 동시에 누르면 화면이 어두워집니다.
  → 마우스로 오류 부분을 드래그하면 캡처됩니다. → 캡처한 그림을 채팅창에 **붙여넣기(`Ctrl`+`V`)**.
- **Mac:** `Cmd`(⌘) + `Shift` + `4` 를 누르고 오류 부분을 드래그. → 바탕화면에 그림 파일로 저장됨.
- **휴대폰:** 평소 스크린샷 방식대로 캡처해서 사진으로 전달.

### 2) 오류 글자(메시지) 그대로 복사하기
- 화면에 글씨로 오류가 보이면, 그 글씨를 마우스로 드래그해서 **복사(`Ctrl`+`C`)** → 채팅에 **붙여넣기(`Ctrl`+`V`)**.
- 글씨가 복사 안 되면 그냥 캡처로 대체해도 됩니다.

### 3) "개발자 도구"의 빨간 글씨 가져오기 (할 수 있으면 좋음, 어려우면 건너뛰기)
오류의 진짜 원인은 여기 숨어 있을 때가 많아요.
1. 오류가 난 페이지에서 키보드 `F12` 를 누릅니다. (오른쪽이나 아래에 복잡한 창이 열림)
2. 위쪽 탭 중에서 **`Console`(콘솔)** 을 클릭합니다.
3. **빨간색 글씨**가 보이면, 그 부분을 드래그해서 복사하거나 그 영역을 캡처합니다.
4. 다시 `F12` 를 누르면 창이 닫힙니다.

### ✅ 클로드에게 보낼 때 이렇게 적어주세요 (복사해서 쓰는 양식)

```
[증상] 예: 로그인이 안 돼요 / 메인이 안 열려요
[언제부터] 예: 오늘 오전 10시쯤부터
[어느 페이지] 예: https://themonster.kr/login
[나만? 다같이?] 예: 휴대폰으로도 안 됨 (다 같이 그런 듯)
[오류 메시지] (캡처 첨부 또는 글자 붙여넣기)
[상태 페이지] Vercel: 초록/빨강 / Supabase: 초록/빨강
[내가 해본 것] 예: 강력 새로고침 함, 시크릿 창 해봄
```

> 이 양식대로만 보내주면, 제가(클로드) 원인을 훨씬 빠르게 찾아드릴 수 있어요.

## 🚦 "내가 해도 되는 것" vs "절대 손대면 안 되는 것"

### ✅ 마음껏 해도 안전한 것
- 새로고침 / 강력 새로고침
- 다른 기기, 시크릿 창, 다른 브라우저로 확인
- 와이파이 껐다 켜기
- 상태 확인 사이트(vercel-status, status.supabase) **보기만** 하기
- 화면 캡처하고 오류 복사하기
- 비밀번호 재설정
- Vercel에서 **이전 초록색 배포로 되돌리기(Redeploy/Promote)** — 증상 A 참고

### ⛔ 함부로 하면 안 되는 것 (반드시 도움 요청)
- Supabase에서 **데이터를 지우거나 수정**하기 ← 🚨 가장 위험! 절대 금지
- Supabase의 설정(Settings), 테이블 구조 바꾸기
- Vercel에서 프로젝트 **삭제(Delete)**, 도메인 설정 변경, 환경변수(Environment Variables) 수정
- 잘 모르는 버튼인데 "정말 하시겠습니까?" 경고가 뜨는 것 → 일단 멈추고 물어보기
- 코드(프로그램 글자) 직접 수정

> 🧠 **기준 한 줄:** "되돌릴 수 있는 것"은 시도해도 되고, **"지우거나 바꾸는 것"은 손대지 말고 물어보세요.**

## 📞 도움 요청 순서 (에스컬레이션)

1. **0단계 점검** → 내 문제인지 먼저 확인
2. **증상별 대처** → 상태 사이트 확인 + 안전한 시도

## 🧰 [부록] 자주 쓰는 컴퓨터 기본 동작

### 브라우저 캐시·쿠키 지우기 (저장된 옛날 정보 청소)
화면이 이상하거나 로그인이 꼬일 때 효과적입니다. (크롬 기준)
1. 키보드 `Ctrl` + `Shift` + `Delete` 동시에 누르기 (Mac은 `Cmd`+`Shift`+`Delete`)
2. 작은 창이 뜨면 기간을 **"전체 기간"** 으로 선택
3. **"쿠키 및 기타 사이트 데이터"** 와 **"캐시된 이미지 또는 파일"** 에 체크
4. **"데이터 삭제"** 버튼 클릭
5. 브라우저를 껐다 다시 켜고 사이트 접속

### 시크릿(비공개) 창 열기 — 깨끗한 상태로 테스트
- 크롬(Windows): `Ctrl` + `Shift` + `N`
- 크롬(Mac): `Cmd` + `Shift` + `N`
- 엣지: `Ctrl` + `Shift` + `N`

### [부록] Vercel 로그 복사하는 법 (여유 있을 때만)
1. https://vercel.com 로그인 → 프로젝트(monster_tree) 클릭
2. 상단 **Deployments** → 맨 위 항목 클릭
3. **"Build Logs"** 또는 **"Runtime Logs"** 에서 **빨간 글씨** 부분을 드래그해 복사하거나 캡처
4. 클로드에게 전달
